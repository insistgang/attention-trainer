## 4. API 客户端分析

`client.py` 中的 `MinimaxAPIClient`:
- 认证方式: Bearer Token (`Authorization: Bearer {api_key}`)
- 请求头: `MM-API-Source: Minimax-MCP`
- Content-Type: `application/json` (无文件上传时)

**已实现的端点**:
1. `/v1/coding_plan/search` - web_search 工具使用
2. `/v1/coding_plan/vlm` - understand_image 工具使用

**错误码处理**:
- status_code 1004: 认证错误 (MinimaxAuthError)
- status_code 2038: 需要实名认证 (MinimaxRequestError)
- 其他: 通用请求错误

## 5. minimax-mcp 可用工具 (共 2 个)

### 5.1 web_search
- 端点: POST /v1/coding_plan/search
- 参数: `query` (str) - 搜索查询，建议 3-5 个关键词
- 返回: 包含 `organic`、`related_searches`、`base_resp` 的 JSON
- 标注: 涉及费用（需注意）

### 5.2 understand_image
- 端点: POST /v1/coding_plan/vlm
- 参数: `prompt` (str) + `image_source` (str, 支持 HTTP URL 或本地文件路径)
- 返回: 图像分析文本描述
- 支持格式: JPEG, PNG, WebP
- 不支持: PDF, GIF, PSD, SVG
- 标注: 涉及费用（需注意）

**服务器信息**:
- 名称: Minimax
- 版本: 1.26.0
- 协议版本: 2024-11-05

## 6. 启动测试结果

### 测试命令:
```
MINIMAX_API_KEY="sk-cp-gc1eL7Zo5imTDlZtYcyOnd28ADoc7J3LZfbc-sR0dGRVvoAAjY7_ZBtCyBh2tzNlww9MYdbZ-TGg9rHgNiJ4rB67"

uvx minimax-coding-plan-mcp
```

### 测试方法:
由于 FastMCP 在 stdio 模式下需要 stdin 输入，使用进程替换来模拟 MCP 握手:

```bash
python -c "from minimax_mcp.server import main; main()" < <(echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{...}}')
```

### 测试结果:
- 服务端成功启动
- 成功响应 initialize 请求
- 服务器返回正常的 JSON-RPC 响应
- 结论: minimax-mcp 可以正常启动并响应 MCP 协议

### uvx 进程立即退出的原因:
- `uvx minimax-mcp` (不带 stdin 输入) 启动后，如果 stdin 关闭则进程退出
- 这是 FastMCP stdio 模式的正常行为 (当 stdin 关闭时，asyncio 事件循环退出)
- 解决方案: 使用进程替换或管道保持 stdin 打开

4. **只暴露了 2 个工具**: web_search 和 understand_image，没有更多端点。
